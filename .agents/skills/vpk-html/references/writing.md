# Writing Rules

The renderer provides structure, not claims. Agents remain responsible for
distilling source material accurately before creating the payload.

- Do not invent metrics, dates, citations, math claims, people, or project
  status.
- Keep headings concrete and short.
- Prefer human-readable captions for diagrams and tables.
- Use `[DATA NEEDED: ...]` only when the agent intentionally wants a visible gap
  marker.
- Avoid filler sections. If material is absent and not intentionally marked, omit
  the optional section.
- For math, order prerequisites before definitions, intuition before formal
  proof, and worked examples before exercises.
